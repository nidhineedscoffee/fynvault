# FinVault Screen Usage

The Stitch screens are the visual source of truth for the product. Use them as a product journey, not as a random gallery.

## Entry

1. `welcome_to_finvault_red_theme`
   Use as the first unauthenticated landing/demo screen.

2. `get_started_finvault`
   Use after the user decides to begin. This is the sign-in/start state.

`welcome_to_finvault` is an alternate landing variant. Keep it available, but do not put it in the primary hackathon flow unless replacing the red-theme welcome.

## Onboarding

3. `create_your_workspace_finvault`
   Use after login to create or name the organization/workspace.

4. `you_re_in_finvault`
   Use as the workspace-created success state.

5. `your_financial_snapshot`
   Use before sync as a baseline snapshot and setup context screen.

## Data Sync

6. `business_discovery`
   Use when FinVault starts learning the business and what systems to connect.

7. `business_discovery_finvault`
   Use for deeper business-model and finance-structure discovery.

8. `syncing_data`
   Use immediately after Zoho, Gmail, or document sync begins.

9. `intelligent_data_syncing`
   Use while data is normalized into structured records.

10. `building_your_ai_cfo_finvault`
    Use while the financial graph, rules engine, and validation layer are being prepared.

## Intelligence Layer

11. `relationship_mapping_finvault_intelligence`
    Use after normalization to show the graph: customers, invoices, payments, vendors, bills, and emails.

12. `customer_intelligence_nexus_global`
    Use as a customer-level intelligence/drill-down view.

## Command Center

13. `financial_health_dashboard`
    Use as the main dashboard after the graph and rules engine are ready.

14. `cash_flow_intelligence`
    Use for cash flow, burn, runway, inflow, and outflow analysis.

15. `risk_center`
    Use for risk triage and alert review.

16. `risk_discovery_intelligence_engine`
    Use when showing automatic risk detection and supporting evidence.

17. `ask_finvault_ai_cfo_executive_briefing`
    Use only after validated calculations exist. This is not a chatbot entry point.

18. `ai_cfo_ready_finvault`
    Use as the final success/readiness state.
